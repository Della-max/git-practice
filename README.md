# Git版本管理实践

## 一、学习资料来源
- Git官方文档：https://git-scm.com/doc
- GitHub官方指南：https://guides.github.com
- 菜鸟教程Git：https://www.runoob.com/git/git-tutorial.html

## 二、实践流程
1. 安装Git并配置全局用户名与邮箱
2. 在GitHub创建远程仓库
3. 使用git clone克隆仓库到本地
4. 完成3次文件修改与提交
5. 使用git push推送到GitHub

## 三、提交记录说明
1. 第一次提交：创建main.py，初始化项目代码
2. 第二次提交：修改main.py，增加功能代码
3. 第三次提交：添加README.md，完成实践报告

## 四、常用Git命令
- git config --global：配置用户信息
- git clone：克隆远程仓库
- git add：添加文件到暂存区
- git commit：提交到本地仓库
- git push：推送到远程仓库
- git log：查看提交记录

## 五、学习心得
通过本次实践，我掌握了Git的基本使用方法，学会了在GitHub上创建仓库、提交代码、查看记录。Git版本管理可以有效记录代码修改历史，方便团队协作与版本回溯，是软件开发必备工具。
